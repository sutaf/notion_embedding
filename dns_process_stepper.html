DNS 동작 과정 — naver.com을 IP로 바꾸기까지

🎯 핵심 요약

  1. 재귀적 질의(Recursive Query): 사용자 PC → Local DNS 까지는 "답을 찾아서 완성해서 줘" 라고 위임
  2. 반복적 질의(Iterative Query): Local DNS → Root/TLD/Authoritative 까지는 한 단계씩 직접 물어봄
  3. 계층 구조: Root(최상위) → TLD(.com, .kr 등) → Authoritative(실제 도메인 관리)
  4. 캐싱(Caching): 한번 조회한 결과는 TTL 시간 동안 Local DNS에 저장돼서, 같은 요청은 1번만에 끝남
